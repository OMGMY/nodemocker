1) 该工具主要使用nodejs基本功能
2）该工具主要用于返回模拟后台数据主要包括card相关的数据，让前端无需等待,目前要慎重提交，每次提交都会创建新的文件，这会导致你需要重新编译，所以建议使用save功能，最后选择提交
3）该工具需要容易增加服务，目前没有过多的设计，app.js为转发器，负责接收请求
4）要使用该工具，需要替换掉安卓代码里面的访问url，该url会在上传或者修改成功后给出，需要注意记录
5) 使用该工具不要随意修改生成的url，否则可能出现不可预知的问题
6）该工具将会逐渐增加代理转发功能，目前可进行http请求代理但还没有实现转发，https还需要解决证书问题
7) 之后在考虑使用session来进行数据通信，现在暂时简单处理
